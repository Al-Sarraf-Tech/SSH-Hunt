# .GEMINI.md - SSH-Hunt

This is a cyberpunk-themed multiplayer game that runs over SSH. The application is a multi-crate Rust workspace, fully containerized with Docker and managed via Docker Compose.

### Key Files & Directories
- `ssh-hunt/`: The root of the Rust workspace, containing all game logic crates.
- `docker-compose.yml`: Defines the main application stack, including the `ssh-hunt` server and a PostgreSQL database.
- `Dockerfile`: A multi-stage Dockerfile that securely builds and packages the Rust binaries.
- `Makefile`: The primary entrypoint for all development and operational tasks.
- `volumes/`: Contains persistent data for the database and game state.

### Local Development Workflow
The entire lifecycle is managed by `make`.
1.  **First-time setup:** Run `make ensure-env` to create a `.env` file from the example.
2.  **Start the server:**
    ```bash
    make up
    ```
    This command builds the Docker image and starts both the `ssh-hunt` and `postgres` services in detached mode.
3.  **Run database migrations:**
    ```bash
    make db-migrate
    ```
4.  **(Optional) Seed the database:**
    ```bash
    make db-seed
    ```
5.  **Connect to the game:**
    ```bash
    ssh -p 24444 <username>@localhost
    ```

### Core Commands
- `make up`: Build and start all services.
- `make down`: Stop and remove all services.
- `make logs`: Tail the logs from running containers.
- `make restart`: Restart all services.
- `make backup`: Create a backup of the game's database and volumes.
- `make restore`: Restore from the latest backup.

### Validation
- **Run all checks and tests:**
  ```bash
  make test
  ```
  This single command runs `cargo fmt`, `cargo clippy`, and `cargo test` for the entire Rust workspace, ensuring code quality and correctness.
