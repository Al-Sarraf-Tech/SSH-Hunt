# Last updated: 2026-03-11 | Evidence: ssh-hunt/Cargo.toml, docker-compose.yml, Makefile, crates/*/src/

## What It Is
Cyberpunk SSH game/MMO. Fully simulated shell and world in Rust.
Publicly playable: `ssh -p 24444 <username>@ssh-hunt.appnest.cc`
Deployed via Docker Compose. Rust workspace lives in `ssh-hunt/` subdirectory.

## Rust Workspace (ssh-hunt/Cargo.toml)

|---|---|---|

## Key Dependencies (ssh-hunt/Cargo.toml)
- `russh` 0.57 — SSH protocol (NOT libssh2 or openssh)
- `sqlx` 0.8 (postgres, tokio-rustls, macros, chrono, uuid, migrate) — DB
- `rhai` 1.20 (sync, metadata) — game scripting
- `tokio` — async runtime
- `clap` 4.5 — admin CLI
- `serde_yaml` 0.9 — game config files

## Docker Compose (docker-compose.yml)
| Service | Image | Port |
|---|---|---|
| postgres | postgres:16 | 5432 (internal) |
| ssh-hunt | built from Dockerfile | 24444 → 22222 |

**Security hardening on ssh-hunt container:**
- `read_only: true` filesystem
- `tmpfs: /tmp` (noexec, nosuid, 64m) + `/run` (16m)
- `cap_drop: ALL`
- `security_opt: no-new-privileges:true`
- `pids_limit: 256`
- `mem_limit: 768m` / `cpus: 1.50`
- `user: 10001:10001`
- `networks: backend` (internal) + `ingress`

## ABSOLUTE SECURITY RULES
1. Player input is handled inside the simulated shell. It must NEVER reach the host OS.
2. NEVER pass player-supplied strings to `std::process::Command` or any shell invocation.
3. `ssh-hunt-server` binary must NEVER run as root.
4. Cloudflare Tunnel handles TLS termination — do NOT add direct TLS to port 24444.
5. Do not weaken `cap_drop: ALL` or `no-new-privileges:true`.
6. Do not add game command logic outside of `shell/` crate.
7. All game state mutations go through `WorldService` — no direct DB writes from shell handlers.
8. `#![forbid(unsafe_code)]` in ssh_hunt_server — do not remove.
